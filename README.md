# Nuclei-Report-Generator

## 📋 项目概述

这是一个基于Nuclei扫描结果的报告生成工具，可以将Nuclei的文本格式扫描结果转换为美观的Word报告。程序支持两种模式：IP地址报告生成和基于设备名称的报告生成。

## ✨ 主要功能

### 1. IP地址报告生成器

- 基于目标IP地址的漏洞统计和详细报告

### 2. 设备名报告生成器

- 基于设备信息（名称、版本、IP）的漏洞统计

## 🚀 快速开始

### 系统要求

- **操作系统**: Windows 7/8/10/11 (64位)
- **内存**: 至少1GB可用内存
- **磁盘空间**: 至少100MB可用空间
- **运行环境**: 无需安装Python或其他依赖
- **查看工具**: 需要Excel或WPS Office查看生成的报告

### 建议目录结构:
```text
项目名称/
├── 原始数据/
│   ├── 目标文件/
│   └── 扫描结果/
├── 生成报告/
│   ├── IP报告/
│   └── 设备报告/
└── 归档/
```

### 安装方式

#### 直接使用打包版本（推荐）

1. 下载 `Nuclei报告生成器.exe` 文件
2. 双击即可运行，无需安装

## 📁 文件结构要求

### 输入文件格式

#### 1. IP地址报告生成器

- **目标列表文件 (.txt)**:
    
    192.168.1.1
    192.168.1.2:8080
    https://example.com
    http://192.168.1.3/path
    
- **Nuclei扫描结果文件 (.txt)**:
    
    [CVE-2023-1234] [http] [high] http://192.168.1.1 ["SQL injection vulnerability"]
    [CVE-2023-5678] [http] [medium] https://example.com ["XSS vulnerability"]

#### 2. 设备名报告生成器

- **设备信息文件 (.txt)** - 制表符分隔:
    
    设备名称1    CentOS 7.9    192.168.1.1
    设备名称2    Windows Server 2019    192.168.1.2
    防火墙设备    FortiOS 6.4    192.168.1.3

- **Nuclei扫描结果文件 (.txt)**:
    
    [vuln-template] [http] [critical] http://192.168.1.1 ["Critical RCE"]
    [info-template] [http] [info] https://192.168.1.2 ["Information disclosure"]

## 🖥️ 使用方法

### 基本使用步骤

#### 步骤1：启动程序

双击运行 "Nuclei报告生成器.exe"

#### 步骤2：选择功能模块

主界面 → 选择需要的功能
├── "IP地址报告生成器" (蓝色按钮)
└── "设备名报告生成器" (绿色按钮)

#### 步骤3：添加文件对

1. 左侧添加目标列表文件（IP模式）或设备信息文件（设备模式）
2. 右侧添加对应的Nuclei扫描结果文件
3. 确保文件数量相同且一一对应

#### 步骤4：设置输出目录

1. 默认输出目录为程序所在目录下的"output"或"device_reports"
2. 点击"浏览"按钮可自定义输出位置

#### 步骤5：开始处理

1. 点击"开始处理"按钮
2. 等待处理完成（进度条显示）
3. 完成后自动打开Excel文件（可选）

## ⚠️ 重要注意事项

### 1. 文件处理注意事项

- ✅ **文件必须一一对应**: 目标文件与扫描文件的顺序必须匹配
- ✅ **文件数量必须相同**: 两侧文件列表数量必须相等
- ✅ **文件格式必须正确**: 确保使用正确的.txt格式
- ❌ **避免中文路径**: 尽量使用英文路径，避免乱码
- ❌ **避免特殊字符**: 文件名避免使用特殊字符

### 2. 程序运行环境

- **首次运行**: 可能需要较长时间启动（5-10秒）
- **杀毒软件**: 首次运行时可能被误报，请添加到白名单
- **管理员权限**: 如果涉及受保护目录，可能需要管理员权限
- **系统区域设置**: 中文系统请确保区域设置正确
